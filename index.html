ASA / CBA Player Requests Hub

Internal use only — Port Adelaide Football Club

🔒Internal Use Only. This portal is strictly for internal PAFC staff. Do not share access or request information outside the organisation.
🤏
What is this hub?
The ASA / CBA Player Requests Hub is a centralised portal for managing AFL and AFLW player appearance requests across Retail and Commercial streams. It tracks player availability, ASA/CBA balance usage, request status, and approval workflows — all in one place and synced live with the team’s monday.com board.
⚠️
Briefing Trent
Trent must still be briefed individually and directly via email for every confirmed appearance — this portal does not automatically notify him. Once a request is approved and confirmed, the relevant approver is responsible for following up with Trent outside of this system. The “Briefed to Trent” checkbox is a manual record only.
✍️
ASA Balance Updates
Jacob is responsible for keeping player ASA/CBA balances up to date. He has a dedicated Balance Updater access PIN which gives him direct access to the Balances tab to update all player percentages and sync them to the board. If you believe a balance is incorrect, please contact Jacob directly.
🔐
Access Levels
Anyone can open and view this hub — no login required to browse. Different actions are unlocked based on your access level:
⚪ ViewerBrowse players, calendar and approved requests. No login needed.
🟡 Retail ApproverView, approve and deny Retail ASA requests.
🟢 Commercial ApproverView, approve and deny Commercial ASA requests.
🪟 Balance UpdaterUpdate player ASA/CBA balance percentages (Jacob).
🔴 AdminFull access — manage players, edit requests, manage PINs.
✉️
How to Submit a Request
Click + New Request in the top navigation. Fill in the request name, type (Retail or Commercial), date, location, time required, players and any relevant notes. Once submitted, the request is automatically routed to the relevant approver and added to the monday.com board. Unapproved requests are only visible to approvers and admins.
✅
Approval Process
When a new request is submitted, the relevant approver will see an amber banner at the top of the hub prompting them to review. Clicking Review & Approve opens a summary of all pending requests with Approve and Deny buttons and an optional comment field. Approved requests become visible to all viewers and are updated on the monday.com board in real time.
🔔
Email Notifications Coming Soon
We are currently working on an automated email notification system so that approvers are notified directly via Outlook when a new request comes through for their stream. In the meantime, approvers should check the hub regularly or will be notified via monday.com bell notifications when a new request is submitted.
📋
monday.com Board Sync
All changes made in this hub — new requests, approvals, balance updates, player edits — are synced live to the ASA / CBA HUB board on monday.com. This means the board always reflects the current state of the hub and can be used as a source of truth for reporting or auditing.
